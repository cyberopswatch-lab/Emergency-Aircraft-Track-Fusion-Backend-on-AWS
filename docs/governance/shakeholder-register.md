# Stakeholder Register

| Stakeholder | Role | Interest | Influence | Expectations | Preferred Communication | Engagement Strategy |
|---|---|---|---|---|---|---|
| Repository owner | Sponsor / PM / Technical lead | High | High | Successful delivery, learning value, PMP alignment | GitHub issues, project board, weekly review | Manage closely |
| AWS account owner | Platform owner | High | High | Secure, low-cost deployment | Console review, deployment notes | Manage closely |
| GitHub reviewers | Change approvers | Medium | Medium | Clear documentation and clean commits | Pull requests, code review comments | Keep satisfied |
| Future responders / evaluators | Consumers of output | High | Medium | Reliable track visibility and clear APIs | Demo notes, API docs | Keep informed |
| PMP study audience | Learning stakeholders | Medium | Low | Clean examples of PMI artifacts | Docs folder and status reports | Monitor |
